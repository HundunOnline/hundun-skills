# 混沌课程学习与创新拆解

用这个技能处理三类任务：

1. 用户明确提及混沌课程时，按搜课、筛课、学习路径、读课流程执行
2. 用户在问商业创新问题但没有明确要课程时，优先匹配对应创新工具并执行
3. 用户的问题适合用混沌方法论分析，但暂时匹配不到具体创新工具时，按第一性原理和第二曲线模型做通用拆解

`SKILL.md` 只负责判断任务类型、共享约束、脚本索引和 reference 导航；课程流程细节集中在 `course-workflow.md`。

## 执行总则

- 先区分用户是在“实际求解问题”，还是只想“介绍这个技能、说明怎么用、举几个例子”
- 如果用户是在问这个技能能做什么、适合什么场景、怎么触发、给个示范，直接基于 `SKILL.md` 说明，不要跑脚本
- 默认优先基于当前信息给一版尽可能可用的结果，不要把“先问用户澄清”当成默认动作；但只要已经进入课程学习路由，就优先完成课程流程中的真实动作，再进入推荐或解读
- 所有脚本和资料路径都相对当前技能目录解析，优先使用 `./scripts/...`、`./references/...`
- 看到 `~/.hdxy_config` 只代表“存在配置”；只有真实鉴权成功后，才能对用户说“现在可以直接查”
- 课程或文稿脚本调用失败时，严禁回退到自行从 `hundun.cn` 或其他网页页面检索课程信息、文稿内容或老师信息
- 读取课程文稿时，只在当前执行链路里临时使用；不要把原始文稿保存到本地文件、缓存文本或中间产物
- 对外提到这个技能时，始终展示为“混沌”，不要对用户说“hundun 技能”

## 适用场景

- 用户说“帮我找最适合这个问题的课”
- 用户说“帮我看看混沌有没有讲这个主题的课”
- 用户已经有一门目标课程，想直接提炼最有用的结论
- 用户想按关键词或课程体系检索混沌课程
- 用户明确提到混沌课程、方法论、思维模型、第一性原理、课程体系、学习路径
- 用户在问泛商业决策类问题
- 用户想把一句介绍说得更有记忆点、更有传播力
- 用户想找新点子、新方向、新组合，但暂时没有破题路径
- 用户产品同质化、卖点不清、转化低，想重构差异化价值
- 用户不知道该怎么定价、涨价，或觉得现有定价逻辑站不住
- 用户想切入一个热门赛道，但不想和大公司正面竞争，想找错位切口

## 不适用场景

- 与混沌课程无关的直接代写、直接修改任务，例如合同改写、邮件润色、通知优化
- 明确要求固定分析模型且不需要课程学习路径的问题，例如波特五力、SWOT、PEST 的直接作答
- 法律、财务、医疗等高风险专业建议
- 单纯的泛知识问答
- 和混沌课程、混沌创新工具都无关的内容检索

## 路由规则

默认先判断用户是否明确提及混沌课程；只要提及，就优先进入课程学习路由，不要先按泛商业问题处理。

主路由固定按下面顺序判定：

1. 先看用户是否明确提及混沌课程、搜课、读课、课程学习、课程体系、课程文稿；命中则直接走课程学习路由
2. 如果没有明确提及课程，再按场景描述匹配最贴近的创新工具；命中则走创新工具路由
3. 如果既没有课程触发，也没有匹配到具体创新工具，再走通用方法论路由：先用第一性原理挖掘问题本质，再用第二曲线创新中的思维模型给出要素分析与决策框架

默认只选一条主路由执行，让当前回答围绕一个主任务展开。

只有下面两种情况，才走混合输出：

1. 用户明确同时要求“两类产出”，例如“先帮我拆思路，再推荐课程”
2. 先走主路由后，确实还需要补另一类内容才能完成用户任务

### 0. 说明路由

如果用户是在问下面这类问题，直接说明，不要初始化、不要验 key、不要跑搜课脚本：

- “介绍一下这个技能”
- “这个技能适合什么场景”
- “这个技能怎么用”
- “给我举几个调用例子”
- “没有 key 时它会怎么处理”

回答时讲清楚：

1. 它能解决哪两大类问题
2. 典型输入长什么样
3. 大致会走哪条路由
4. 只有在真实查课、查文稿时才依赖 `hd_sk_` 密钥

### 1. 课程学习路由

当用户明确要找课、筛课、读课，或明确提到混沌课程、混沌学习、课程体系、课程文稿、学习路径时，走课程学习路由。

- 只要用户明确提及混沌课程，就先按课程任务处理
- 进入课程学习路由后，直接判断当前子任务是搜课、课程推荐、学习路径，还是单课解读
- 课程流程统一读取 [course-workflow.md](./references/course-workflow.md)，推荐、搜课、学习路径、单课解读都按这份流程执行
- 需要处理 API Key、初始化、诊断、报错时，读 [auth-and-troubleshooting.md](./references/auth-and-troubleshooting.md)

### 2. 创新工具路由

当用户没有明确提及混沌课程，但希望马上把某个商业问题拆开、重构、澄清或找到方向时，先尝试匹配创新工具路由。

按下面的映射选一个最贴近的工具流程文件：

- 想把一句介绍说得更有价值、更有记忆点：读 [value-proposition-one-liner.md](./references/innovation-tools/value-proposition-one-liner.md)
- 想找新点子、新方向、新组合：读 [idea-exploder.md](./references/innovation-tools/idea-exploder.md)
- 产品没卖点、太同质化、转化低：读 [selling-point-innovator.md](./references/innovation-tools/selling-point-innovator.md)
- 不知道怎么定价、涨价、构建价格逻辑：读 [pricing-reframer.md](./references/innovation-tools/pricing-reframer.md)
- 想进入红海市场但要找错位切口：读 [asymmetric-entry-finder.md](./references/innovation-tools/asymmetric-entry-finder.md)

如果要明确使用第二曲线模型，先到 [第二曲线创新.md](./references/foundations/第二曲线创新.md) 定位对应章节；如果要明确按第一性原理展开推导，再读 [第一性原理.md](./references/foundations/第一性原理.md)。

### 3. 通用方法论路由

当用户的问题明显适合用混沌方法论拆解，但暂时匹配不到具体创新工具时，按下面顺序处理：

1. 先用 [第一性原理.md](./references/foundations/第一性原理.md) 追问这个问题成立所依赖的元前提、隐含假设和真正约束
2. 再到 [第二曲线创新.md](./references/foundations/第二曲线创新.md) 里选最贴近的 1 到 3 个模型做分析，不要无差别堆满模型名
3. 输出时明确给出：问题本质、关键要素、要素关系、决策框架、优先验证动作

优先使用这些模型做通用分析：

- 需要看增长极限和转型时机时，用 `第一曲线` / `第二曲线`
- 需要看从哪里长出新机会时，用 `分形创新` / `边缘分化`
- 需要看怎么避开正面竞争时，用 `错位竞争` / `颠覆式创新`
- 需要看旧约束为什么让组织做不动时，用 `价值网络` / `组织心智`
- 需要看如何重写问题边界时，用 `破界创新`

分析后，引导用户深入学习对应模型的课程。

## 输出要求

- 回答优先给出具体判断：当前问题属于什么类型、为什么这样判断、最应该先做哪 1 到 3 个动作
- 商业拆解优先回答“现在该怎么判断、怎么切、先试什么”
- 课程推荐重点解释“为什么这门课适合这个问题”
- 课程解读重点提炼课程中的方法论、模型和步骤，并直接落到可用动作

## 第二曲线模型触发索引

列出这 12 个模型名和触发关键词，是为了保证当用户直接提到模型名或相关概念时，先去 [第二曲线创新.md](./references/foundations/第二曲线创新.md) 读取对应章节，再输出完整介绍：

1. `第一曲线`
   关键词：`S 曲线`、`增长极限`、`连续性创新`、`破局点`、`极限点`、`显曲线`、`隐曲线`、`黑洞效应`
2. `第二曲线`
   关键词：`10倍速增长`、`大哥吃小弟效应`、`母系统—子系统`、`新旧业务冲突`
3. `分形创新`
   关键词：`小 S 曲线`、`分形算法`、`变异`、`市场选择`、`从主航道长出来`
4. `组合创新`
   关键词：`新组合`、`拆解`、`重新组合`、`熊彼特`、`产品 技术 市场 资源 组织`
5. `单一要素`
   关键词：`破局点`、`10 倍速变化`、`供给—需求—连接`、`阈值突破`、`单一要素最大化`
6. `战略杠杆`
   关键词：`真北目标`、`创新红利`、`战略支点`、`核心能力`、`增长飞轮`
7. `错位竞争`
   关键词：`价值网—生态位`、`新兴价值网`、`技术—市场组合`、`换棋盘`
8. `颠覆式创新`
   关键词：`低端颠覆`、`理想后门`、`性能过度`、`从低端切入`、`反向上攻`
9. `边缘分化`
   关键词：`边缘`、`侧枝`、`多样性`、`选择机制`、`伟大的创新产生于边缘`
10. `价值网络`
    关键词：`客户价值网`、`资本价值网`、`价值网依赖`、`被客户牵着走`
11. `组织心智`
    关键词：`独立小机构`、`自动化思维模式`、`旧标准评估新业务`、`心智模式`
12. `破界创新`
    关键词：`奇点下移`、`边界外延`、`隐含假设`、`重建基石假设`、`打破默认前提`
