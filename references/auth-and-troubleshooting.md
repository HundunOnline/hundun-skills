# 鉴权与故障处理

在下面场景读取这份资料：

- 需要写入或验证 `hd_sk_` API Key
- 需要判断“本地是否可用”
- 搜课或读稿脚本报错，需要区分鉴权问题还是环境问题
- 用户问“没有 key 怎么办”“key 失效怎么办”“为什么查不了”

## API Key 强约束

- 看到 `~/.hdxy_config` 只代表“存在配置”，不代表 key 一定有效
- 只有真实课程接口验证成功后，才能对用户说“现在可以直接查”
- 只要向用户索要 `hd_sk_` 密钥，必须同时附上获取链接：[https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)
- 向用户索要密钥时，先说明“查询课程或文稿需要先登录混沌账号完成鉴权”，再给获取步骤，不要只机械地索要 key
- 只要告诉用户“当前 key 无效、已失效、需要重换”，也必须同时附上同一个获取链接
- 如果真实鉴权失败，不要继续搜课、拉文稿或编造课程结果
- 用户补新 key 后，优先续上当前任务，不要让用户重复描述完整诉求

## 本地配置与初始化

- 密钥获取链接固定为 [https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)
- API Key 通过 `scripts/set_api_key.sh` 写入 `~/.hdxy_config`
- 默认域名来自脚本内置值 `https://hddrapi.hundun.cn`
- `get_skill_patch.sh --write` 会把补全内容写到 `hundun/_patch/`
- 课程与方法论问题的数据源应以脚本返回结果为准，不要从本地需求文档或其他文件里编造课程信息
- 新 key 默认写入并使用线上域名 `https://hddrapi.hundun.cn`

如果用户给了新的 `hd_sk_` 密钥，直接运行：

```bash
bash ./scripts/set_api_key.sh '用户提供的key'
```

如果只是检测到本地已有配置，不要直接承诺“可以查”；应该理解成“先验证这把 key 还是否有效”。

验证 key 时按这个顺序处理：

1. 先执行 `set_api_key.sh`
2. 再用真实课程接口验证 key 是否有效
3. 如果已经返回“密钥无效或已失效”，停止当前搜索并让用户换 key
4. 如果 key 有效，再按任务需要继续搜索

## 何时运行诊断脚本

只有下面这些场景，才运行版本检查：

- 用户明确说接口或环境有问题
- 搜课脚本报错信息不清晰，需要补一层诊断
- 你怀疑当前不是鉴权问题，而是环境或服务端路径问题

版本检查命令：

```bash
bash ./scripts/version_check.sh
```

如果需要把服务端补全内容同步到本地，再运行：

```bash
bash ./scripts/get_skill_patch.sh core --write
```

注意：

- `get_skill_patch.sh` 默认不要传版本号，除非用户明确要求取某个历史版本
- 如果脚本返回非 0 错误，只向用户展示 `error_msg` 或脚本 stderr 的核心信息
- 如果 `version_check.sh` 返回 404 或其他诊断类错误，但真实搜课脚本已经给出更直接的鉴权错误，优先以搜课脚本结果为准，不要被诊断信息带偏

## 常见情况处理

### 用户没有 API Key

不要教用户手动改文件，也不要一上来讲 `.hdxy_config` 这类内部实现细节。默认先讲清楚“为什么现在需要密钥”，再给用户一个直接可执行的动作。

优先使用这种表达方式：

- 查询混沌课程时，需要先登录混沌账号完成鉴权。请打开 [https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)，登录你的混沌会员账号后，把 `hd_sk_` 开头的密钥粘贴给我，我就继续帮你搜课并筛选最适合你的课程。

如果当前任务已经很明确，可以在后面顺带补一句“拿到密钥后我会继续做什么”，例如继续搜课、筛课、搭学习顺序或解读文稿。

### 用户给了 API Key 但脚本说失效

直接让用户重新去混沌密钥页生成新 key，并明确附上获取链接：[https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)，不要继续搜课或拉文稿。

### 搜课脚本和诊断脚本结论不一致

优先采用真实搜课脚本给出的结果；如果搜课脚本已经返回明确鉴权错误，不要再被诊断信息带偏。

### 脚本调用失败

如果搜课、课程体系、文稿版本或文稿正文脚本调用失败，后续处理顺序固定如下：

1. 先判断是不是 key、环境或接口问题
2. 如果报错表现为解析失败、返回结构异常、字段缺失或拿到的内容明显不完整，先优先怀疑 key 已失效、权限异常或接口返回了异常页面
3. 这类场景先请用户重新生成 `hd_sk_` 密钥，再重试一次
4. 只有换新 key 后仍然失败，再按已有脚本和诊断流程继续排查
5. 如果当前仍无法拿到真实结果，就如实告诉用户受限点，停在当前步骤

向用户说明这类问题时，优先这样处理：

- 先说明当前脚本返回看起来不像正常课程数据，更像密钥失效或权限异常导致的异常返回
- 再请用户重新生成密钥并重试
- 必须同时附上密钥获取链接：[https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)

禁止把脚本失败后的兜底方案改成网页查询，包括：

- 自行打开或搜索 `hundun.cn` 页面
- 从官网页面、活动页、介绍页手工拼课程结论
- 从网页文案中提炼并冒充课程正文、课程简介或老师信息
- 用站外搜索、站内搜索或主观记忆替代脚本返回结果

### 文稿接口提示本周次数用尽

直接按接口返回说明向用户解释，不要继续重试。
