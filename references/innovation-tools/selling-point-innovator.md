# 产品卖点创新器

当用户的产品或服务看起来和别人差不多、卖点不清、转化不高，想重构差异化表达和可测试卖点时，使用这个流程。

## 适用问题

- “我们的产品没卖点，广告打不动”
- “大家都在说差不多的话，我怎么说得不一样”
- “用户看了页面也不觉得非买不可”

## 不适用问题

- 用户只要写一篇完整投放文案
- 用户没有提供任何产品、用户、场景信息

## 最少输入

- 产品或服务是什么
- 核心用户是谁
- 当前购买或转化卡在哪

如果缺信息，最多补 3 个问题：

1. 用户现在为什么不立刻买
2. 你的产品最常被拿来和谁比较
3. 用户最在意的是结果、效率、风险还是身份认同

## 执行流程

1. 先拆出现有表达里最普通、最容易同质化的部分
2. 找出用户真正购买的不是“产品本身”，而是哪个结果、风险降低或身份感
3. 用第一性原理重写价值单元：用户付钱到底是为哪一个关键变化
4. 从跨界类比、使用场景切换、反共识承诺、结果导向表达里，生成 3 条不同卖点方向
5. 每条卖点都补一个更适合测试的表达角度：
   - 页面标题怎么写
   - 广告首句怎么写
   - 销售开场怎么说
6. 选出最值得优先验证的 1 条卖点，并说明为什么
7. 给出一个最小测试建议，不做复杂投放方案

## 输出结构

1. 当前产品表达的问题在哪里
2. 3 条差异化卖点方向
3. 每条卖点适合打哪类用户、哪类场景
4. 推荐先测的 1 条卖点
5. 下一步测试动作

## 当前版本的方法要求

- 卖点必须围绕用户结果来表达，不要只讲功能堆叠
- 第二曲线里优先使用这些具体模型：
  - `组合创新`：重组产品、场景、结果和表达角度
  - `单一要素`：找出最该被放大的那个关键卖点
  - `错位竞争`：把卖点建立在“为什么选你而不是替代方案”上
  - `破界创新`：当行业卖点都说一样的话时，改写默认比较方式
- 使用前直接到 [第二曲线创新.md](../foundations/第二曲线创新.md) 里找对应章节并补读
- 如果需要明确按第一性原理展开，先追加阅读 [第一性原理.md](../foundations/第一性原理.md)
- 可以借用混沌的模型思路，但先以“用户为何选择你而不是替代方案”为主线
- 输出里要兼顾启发性和可试错性
