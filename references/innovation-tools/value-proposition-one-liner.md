# 一句话价值提亮器

当用户已经有一段关于自己、公司、产品或项目的介绍，但表达平、长、散、记不住时，使用这个流程。

## 适用问题

- “帮我把这段介绍说得更有记忆点”
- “我们到底该怎么一句话介绍自己”
- “这段官网标题 / 电梯介绍 / 小红书简介太普通了”

## 不适用问题

- 用户要完整品牌故事、整页文案、广告投放方案
- 用户根本还没想清楚自己卖什么、解决谁的问题

## 最少输入

- 当前介绍原文
- 目标对象是谁
- 希望对方记住什么

如果缺关键信息，最多补 3 个问题：

1. 你最想打动的是谁
2. 你解决的最关键问题是什么
3. 你和常见替代方案最大的不同是什么

## 执行流程

1. 先提炼原句中的对象、问题、结果、差异点
2. 用第一性原理拆掉空话、套话、内部黑话，只保留用户真正在意的价值
3. 用“旧替代方案为什么不够好”反推一句话里必须出现的张力
4. 至少给出 3 个版本：
   - 稳健清晰版
   - 有冲击力版
   - 适合对外传播版
5. 每个版本都说明“为什么这句话更容易被记住”
6. 最后给出 1 版推荐主句和 1 句使用建议

## 输出结构

1. 推荐的一句话主表达
2. 备选表达 2 到 3 条
3. 每条表达分别击中了什么价值
4. 适合放在哪些场景里使用

## 当前版本的方法要求

- 优先用“对象 - 痛点 - 结果 - 差异”四段式来重写
- 第二曲线里优先使用这些具体模型：
  - `组合创新`：重组表达中的对象、问题、结果与差异
  - `错位竞争`：强调你和常见替代方案到底有什么不同
  - `破界创新`：当原有介绍建立在错误假设上时，直接改写问题边界
- 使用前直接到 [第二曲线创新.md](../foundations/第二曲线创新.md) 里找对应章节并补读
- 如果需要明确按第一性原理展开，先追加阅读 [第一性原理.md](../foundations/第一性原理.md)
- 可以借用第二曲线、第一性原理和其他成熟模型的视角，但先不要堆理论术语
- 目标是让表达更准、更短、更有张力，不是追求华丽文风
